the premise

What if software built software?

An order goes in: “build this product.”

Research, specs, design, code, review — a finished PR comes out.

No org chart. No standups. Almost no humans.

Corellia is a design for an autonomous AI software factory — the whole product process, not just the typing-code part.

the machinery

One move, repeated

receive a goal → decide → emit the result

decide = do it yourself, or split it into smaller goals

                "build this product"
                        │
        ┌───────────────┼───────────────┐
    research          write-prd       build
                                        │
                              ┌─────────┴─────────┐
                          implement           implement

Everything is a typed goal — a PRD, a diff, a renamed variable. The same move runs at every depth of the tree.

the machinery

A goal-type is a harness

Every kind of goal comes pre-rigged:

context · memory · tools · checks · model size

A goal can only touch what its type grants.

No job titles exist. "The design phase" is just the region of the tree where design-typed goals got spawned. Roles are emergent.

quality control

Who checks the work?

split — was the plan sound and complete?

output — does each piece meet its contract?

integration — does the whole satisfy the original ask?

Fail a check → retry on a bigger model. Keep failing → summon a human. One loop ties quality, cost, and people together.

the people

Humans enter at exactly three gaps

competence — every model tier tried, and failed

authority — consequences outrun what any check can underwrite

physical — sign the contract, tap the real device

The human is an exception handler, not a participant. All three gaps are rare by design — and designed to shrink.

the machinery

The factory remembers

project memory — facts about this codebase

type memory — how this operation is done well, everywhere it ever ran

the event log — everything that ever happened, append-only

Memory is a projection of the log. A critique-type that has run 10,000 times holds wisdom no single project could teach it.

interlude · hofstadter, 1979

Strange loops, briefly

Climb a hierarchy, level by level — and find yourself back where you started. The bottom reaches up and rewrites the top.

— the idea at the heart of Gödel, Escher, Bach

Escher's two hands drawing each other. Gödel's sentence that talks about itself. Hofstadter called it a tangled hierarchy.

the thesis

The recursion is not the loop

Inside one run, the hierarchy is deliberately tame:

splits shrink · budgets bound · parents verify children — never the reverse

Every tree terminates. A fugue that resolves.

No tangled levels inside a run: work flows down, verdicts flow up, and a leaf can never overrule its parent. The strange loop lives somewhere else — across runs.

Drawing hands

   harness ──── spawns ────▶ goals ────▶ leaf hits friction
      ▲                                          │
      └─────────── improvement PR ◀──────────────┘

A leaf's friction becomes a PR that rewires the harness producing future leaves.

The product of the machine edits the machine.

This is the "evolve" kind of goal — the factory changing itself. Escher's hands: each one drawing the other.

level-crossing 2 of 4

Instances edit their class

Every run of critique leaves a lesson behind.

Those lessons rewrite how the next critique behaves.

The students, collectively, rewriting the course.

Type memory: experience flows from the level of instances up to the level of their own kind — a crossing object-oriented programmers are taught never to allow.

level-crossing 3 of 4

Moves become rules

A split that keeps recurring gets promoted into a pattern — a standing rule for how to split.

Moves of the game, promoted into rules of the game.

Gödel's trick, operationally: behavior encoded as data, and the data then governs behavior.

level-crossing 4 of 4

The log is a self-symbol

The factory's record of itself is causally active.

Track record → earned trust → wider permissions.

What it knows about itself changes what it may do.

Hofstadter argued a self is a self-model that feeds back into behavior. The event log is exactly that — and it decides which acts the factory is trusted with.

the leash

A human sits at every crossing

improvement PRs — human-reviewed, always

patterns — trusted only on human signoff

memory — eval-gated in, decays out

no factory-factory — the architecture itself is locked

The loop is real, and leashed. Nothing crosses a level without passing a gate — and the deepest level only changes by human hand.

the rhyme

Gödel’s hard hat

No system can prove its own consistency from inside.
— second incompleteness theorem, 1931

The factory's version: some acts’ consequences outrun every internal check. For those, it must summon a human.

That's the authority gap — the incompleteness theorem wearing a hard hat. Not a missing feature; a mathematical posture.

the self

Where does the factory live?

Trees are ephemeral — spawned, finished, gone.

What persists: type memory, patterns, harness code, the log — the substrate the loop carries forward.

The strange loop is the factory’s self.

Like cells in a body: every worker is replaced, yet something with a character and a track record endures — the thing that loops.

the loop, fed back

The analysis changed the design

We described the factory in Hofstadter’s terms — then mined his books for more.

Four ideas made it back in as concrete mechanisms.

The analysis crossed levels and rewrote the thing it described — a small strange loop of its own.

The deck so far read the design through GEB. This section is the reverse crossing: GEB editing the design. Four borrowings follow.

fed back · 1 of 4

Not a sphex wasp

The sphex repeats its routine forever, never noticing the loop.

Old design: fail → retry at a higher tier — the same mistake with a bigger brain.

Now: retries see their failures, re-splits perturb the failed split, and a repeated failure signature jumps out of the ladder early.

Attempt N+1 carries attempt N's artifact and verdict; an isomorphic failure means the ladder isn't working — escalate hard or block. The budget is the mechanical exit; noticing the loop is the intelligent one.

fed back · 2 of 4

Quoted, never obeyed

GEB’s use/mention distinction: using a sentence vs. quoting it.

Injected memories are quoted data, never directives — evidence the parent mentions, never an instruction the child obeys.

Memory poisoning becomes a data-quality problem, not prompt injection.

Memories arrive provenance-labeled, attributed, and structurally segregated from the harness's own instructions. The philosopher's distinction turns out to be a security boundary.

The terraced scan

        novel shape · no memo
                  │
      ┌───────────┼───────────┐
   split A     split B     split C      k drafts, cheap tier
      └───────────┼───────────┘
             judge ranks
                  │
               split B ──▶ deepened, full tier

Don’t bet the subtree on a single draw: candidates compete, only the winner is deepened.

Architecture runs the same scan — the losers become the ADR’s “alternatives considered,” actually explored, not retrofitted.

From Copycat: commitment rises as confidence rises. Exploration cost stratifies by uncertainty, like every other cost in the design.

fed back · 4 of 4

What calibrates the calibration?

GEB’s Tortoise: justification inside a system regresses forever.

The cut: ground truth is exogenous — merged PRs, production outcomes, human verdicts.

Never another eval.

Judges are calibrated by golden sets; golden sets come from where the factory meets the world. There is no eval-of-eval-of-eval — the regress terminates outside the system.
