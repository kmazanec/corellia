# Ride-along items (small, any iteration)

## Problem
A set of small, individually-tiny enhancements were parked in the roadmap as
"ride-along items (small, any iteration)" — none large enough for its own iteration,
none yet built. They are collected here so they are not lost when the roadmap is
deleted:

- **Pattern-trust ceremony CLI** — a CLI surface for the pattern-trust ceremony.
- **Risk-sensitivity segment matching** — segment matching for risk sensitivity.
- **pgvector / relevance retrieval** — relevance retrieval over the substrate
  (related to the deferred "semantic retrieval" non-goal).
- **Decision-brief notification surface** — a surface to notify on decision briefs.
- **Replay harness over the event log** — replay tooling over the event log
  (related to the deferred "full replay tooling" non-goal).

(The roadmap noted decide-phase metering already graduated into F-35, so it is not
listed here.)

## Evidence
the roadmap (now folded into docs/iterations/index.md + docs/issues/) "Ride-along items (small, any iteration)" section. The roadmap is
being deleted; these bullets were future/unbuilt.

## Proposed direction
Each is small enough to ride along with an unrelated iteration when convenient.
Split out into its own issue if/when one grows or is scheduled.

## Acceptance hint
Each ride-along item is either built into some iteration or promoted to its own
tracked issue; none is silently dropped with the roadmap.
